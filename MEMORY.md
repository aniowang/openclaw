## 2026-02-12 GitHub API 成功經驗 (新增)

**情境：** 使用者想要連結 GitHub 並上傳檔案，測試兩種方法：
- Method 1: Git CLI → ❌ 失敗（`git` 不在 PATH）
- Method 2: GitHub REST API → ✅ 成功

**關鍵發現：**
1. `gh` CLI 不在 PATH 中，無法執行 Git 命令
2. Token 必須有 `repo` 權限
3. 使用 `PUT /repos/{owner}/{repo}/contents/{path}` 上傳檔案
4. 檔案內容需用 Base64 編碼
5. 成功回應狀態碼為 201

**測試檔案：**
- `method_command.txt` → 意圖用 Git CLI 上傳（失敗）
- `method_api.txt` → 用 API 上傳（成功）

**學到的教訓：**
- GitHub API 是 `gh` CLI 的可靠替代方案
- 在 CLI 工具不可用的環境下，REST API 是唯一選擇
- 兩個測試檔案都保留在 mcp_test repo 中作為參考
